Document 1 — Situation Report
What Your Senior Actually Asked For vs. What Got Built
Your senior's two Markdown specs are, honestly, fine. They name the right prior work (Lazy Prices, Tetlock, PEAD, Hoberg & Phillips), they set sensible scope, they flag the right risks. The specs are not the problem. The problem is in implementation decisions that were made silently during the build, most of which drift away from what the specs actually said.
Concretely, here is the gap between spec and reality:
The Quiet Change spec says to use the qualitative section of 決算短信 (kessan tanshin — Japanese earnings-summary filings) and benchmark CAR against TOPIX. The current code instead benchmarks against the Nikkei 225, uses whatever text is in TDnet/EDINET without filtering by filing type, and — most critically — produced a flagged pick (3923 ラクス) where the "narrative change" is really a mismatch between an annual filing and an interim filing. The spec's risk section does not anticipate this failure mode, but it is the single biggest driver of the current output being untrustworthy.
The Similar Company spec explicitly warns about over-fitting to a small template set and about erroneous matches across dissimilar contexts. The current code uses the literal three templates named in the spec as examples (Resona, YA-MAN, World) as if they were the complete template library — which the spec never intended. The spec also lists Hoberg-Phillips as a recommended supplementary measure for peer filtering; this is not implemented at all.
The aggregation step isn't really specified in either document. The current code takes the intersection of both detectors, which is the crudest possible approach and throws away borderline-strong candidates in either dimension.
The backtest step is explicitly written into both specs as a 1–2 week task. It has not been performed. Without it, nothing about the current output can be trusted, including the sign of the signal (Lazy Prices found changers underperform, so your system needs evidence it has inverted that relationship, not assumptions).
What the Current Output Tells Us
The report you ran produced one high-conviction pick, 3923 ラクス, and that pick's own explanation makes clear the comparison was between an annual filing and a half-year interim filing — structurally different document types that will always differ. This is not a real Quiet Change; it is a data-pipeline artifact. The Similar Company output has almost every match clustered at 80–100% phase progress, which either means every flagged candidate has already completed its transformation (no upside remaining) or the partial-DTW phase calculation is miscalibrated. Either way the "phase progress" column cannot currently be used for decisions.
Verdict
The system is closer to a working prototype than to a production research tool. The core ideas are defensible and aligned with published literature. The implementation has four foundational bugs (document-type mismatch, wrong benchmark, weakest-variant CAR, tiny template set) and is missing a backtest. Until those are fixed, additional sophistication — more features, more LLM calls, better aggregation — will just make a bad foundation more expensive.
