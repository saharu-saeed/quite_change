# R+ Classification View — Quiet-Change Clean Build

This folder is the **self-contained, clean** version of the R+ (revenue-up) Japanese-equity classification deliverable. Everything required to rebuild the HTML output ships in this directory. There are no external dependencies on the surrounding project tree.

## What this is

A two-tab interactive HTML view classifying 82 Japanese IT-sector companies for FY 2025 into:

- **R+ × S-** (revenue up, stock down — the focus tab) — 41 companies across 5 reason buckets
- **R+ × S+** (revenue up, stock up — secondary tab) — 41 companies across 8 reason buckets

Every company card has three layers:
1. **Genuine research** (green block) — Round 2 verified JP web-search summary with EN+JP toggle and source attribution
2. **Pattern** (collapsed sub-line) — bucket-templated label
3. **Source prose** (collapsed sub-line) — fixture-time prose (EN original + JP auto-generated summary)

## What each file does

### `build/build_view.py`

A single-file Python script with five logical sections:

1. **Path layout** (top): defines `HERE`, `ROOT`, `DATA`, `PARSER_OUTPUTS`, `DELIVERABLES` constants so the script knows where to read inputs and write outputs.
2. **Data loaders**: read `company_research_2025.json`, `it_census_universe.csv`, and every `parser_output_*.json` file.
3. **Classifier** (`reason_bucket` function + `TICKER_OVERRIDE_S_MINUS` / `TICKER_OVERRIDE_S_PLUS` dicts): maps each company-year record to a reason bucket using a mix of rule-based and override-based logic.
4. **Bucket definitions** (`BUCKET_LABELS` dict): plain-language title + note for each bucket in both English and Japanese, structured as "In plain terms / Rule used / Why this matters / What to watch."
5. **HTML / CSV renderer**: emits the deliverable HTML with embedded CSS + JavaScript (language toggle, tab switching, collapsible buckets), and a parallel CSV export.

### `data/company_research_2025.json`

84 ticker keys → `{name, jp_summary, en_summary, source_hint}`. Each company has:
- `jp_summary` — Japanese narrative (1500–3500 chars typical) with "What happened / Why / Stock reaction / Structural drivers / Risks" structure, plus a "Round 2 verification additions" addendum block at the end (which adds source-grounded specifics from the post-write verification pass).
- `en_summary` — English mirror of the same narrative.
- `source_hint` — comma-separated list of sources consulted (Nikkei TDnet, kabutan, minkabu, logmi, IR materials, etc.).

### `data/it_census_universe.csv`

Census registry of ~122 IT-sector Japanese tickers with `ticker`, `company_name`, `sub_type`, `size_band` columns. Used by the build script to look up display names and badge information.

### `data/parser_outputs/parser_output_*.json`

Per-company fixture files. Each has a `records` array, one entry per year, with structured fields: `ticker`, `year`, `業績` (business sign ◎/○/△/✕), `株価` (stock sign), `revenue_dir`, `op_dir`, `net_dir`, `stock_yoy_pct`, `op_abs`, `net_abs`, and a `rawExplanation` text. The build script reads ALL of these and selects the records where `year == 2025` for the view.

### `audit/verification_findings_round2.md`

R+×S- Round 2 audit trail. For every one of the 41 R+×S- companies, lists what the Round 2 verification web search confirmed (✅), what new facts were discovered (➕), and where the original entry's inferences were wrong or unsupported (⚠️).

### `audit/verification_findings_round2_RplusSplus.md`

Same structure as the file above but for the 41 R+×S+ companies.

### `deliverables/R_PLUS_2025_view.html`

The deliverable. Open it in a browser. Features:
- Top-right EN / 日本語 language toggle
- Two tabs (R+×S- default-selected, R+×S+ secondary)
- Each tab has reason-bucket toggles (click ▸ to expand)
- Each company card has the green Genuine-research block (primary), a collapsed Pattern line, and a collapsed Source-prose toggle

### `deliverables/R_PLUS_2025_view.csv`

CSV mirror with one row per company-year: `name, ticker, group, reason_label, revenue_dir, stock_pct_yoy, genuine_research_en, genuine_research_jp, genuine_source_hint, one_line_reason_templated, sub_type, size, year`.

### `deliverables/OPENING_REMARKS_2026-06-04.md`

5-line opener + verified-fact talking points + audit-trail framing for the 2026-06-04 meeting.

## Classification logic in plain terms

**R+ × S-** uses 5 buckets:

| Bucket | Companies | Selection rule |
|---|---|---|
| Durably overlooked | 3 | 2+ prior years tagged "Q2" (= 3+ years of business-up / stock-down) |
| All financials up, stock still fell | 2 | revenue+OP+NP all up AND stock YoY < -30% |
| Revenue up, profit compressed | 17 | 業績 sign = "mixed" (revenue up but operating or net profit down) |
| Loss-making SaaS de-rated | 4 | ticker override: Money Forward, freee, JMDC, Appier |
| Orphan — no analyst coverage | 4 | ticker override: HENNGE, Cybozu, TechMatrix, JAPAN MATERIAL |
| AI disrupted | 3 | ticker override: VisasQ, HEROZ, NSD |
| Mature giant drifting | 3 | ticker override: MIXI, Kakaku.com, LY Corporation |
| Special situation | 5 | ticker override: TerraSky, Remixpoint, coconala, Secure, Trend Micro |

**R+ × S+** uses 8 buckets:

| Bucket | Companies | Selection rule |
|---|---|---|
| Sustained compounder | 1 | 3+ consecutive Q1 years (all-three-up + stock-up) |
| Recovery caught up | 9 | prior 1-2 years had a Q3 or Q4 (decline year) |
| Acceleration + re-rating | 6 | all three financials up AND stock YoY > +30% |
| Clean grower | 14 | all three financials up, stock up but not 30%+ |
| Activist / corporate-action catalyst | 2 | ticker override: Fuji Media, SoftBank Group |
| Profitability inflection | 3 | ticker override: Rakus, Asteria, BIPROGY |
| Inbound / demographic tailwind | 3 | ticker override: istyle, Sharing Tech, Smaregi |
| Sector-specific tailwind | 3 | ticker override: Toho, TV Tokyo, SKY Perfect JSAT |

The rule-based buckets use the actual financial direction and stock return fields. The ticker-override buckets are anchored to the Round 2 verification work (see `audit/`) and assign a company based on its documented dominant 2025 driver.
